## 🤔 바이브코딩이란?

**Vibe Coding (바이브코딩)**
= 분위기(Vibe)에 맞춰 자연스럽게 코딩하는 방식
= AI와 함께 대화하듯 개발하는 방법

### 핵심 철학
```
❌ 코드부터 짜기

❌ 머릿속으로만 생각

❌ 한 번에 완성하려고 함
✅ 작게 나눠서 하나씩 완성
```

## 1️⃣ PRD (Product Requirements Document)

### 뭔가요?
**제품 요구사항 문서** = "뭘 만들지 정리한 문서"

### 왜 필요해요?
- 만들다가 "이게 뭐였지?" 방지
- 다른 사람과 공유 가능
- AI한테 설명하기 쉬움

### PRD에 들어갈 내용

```markdown
# [프로젝트명] PRD

## 1. 개요
- 프로젝트명: 
- 한 줄 설명: 
- 만드는 이유: 

## 2. 타겟 사용자
- 누가 쓸 건가요?
- 어떤 문제를 해결해주나요?

## 3. 핵심 기능
- 기능 1: 
- 기능 2: 
- 기능 3: 

## 4. 성공 기준
- 이게 되면 완성!
- 예: "타자 연습 10문장 완료 가능"

## 5. 제외 범위 (안 만들 것)
- 이건 나중에:
- 이건 안 함:
```

### 예시: 시니어 타자 연습 PRD

```markdown
# 시니어 타자 연습 PRD

## 1. 개요
- 프로젝트명: 시니어 타자 연습
- 한 줄 설명: 어르신을 위한 쉽고 큰 글씨 타자 연습
- 만드는 이유: 기존 타자 연습은 글씨가 작고 어려움

## 2. 타겟 사용자
- 55세 이상 컴퓨터 초보자
- 키보드 타이핑이 서툰 분
- 문제 해결: 천천히, 큰 글씨로 연습 가능

## 3. 핵심 기능
- 기능 1: 난이도 선택 (쉬움/보통/어려움)
- 기능 2: 큰 글씨 문장 표시
- 기능 3: 정확도/속도 측정
- 기능 4: 격려 메시지

## 4. 성공 기준
- 10문장 연습 완료 가능
- 정확도/타수 표시
- 모바일에서도 동작

## 5. 제외 범위
- 회원가입/로그인 (나중에)
- 랭킹 시스템 (나중에)
- 영어 타자 (안 함)
```

## 2️⃣ 기술 설계 (Tech Design)

### 뭔가요?
**"어떻게 만들지"** 기술적인 계획

### 들어갈 내용

```markdown
# 기술 설계

## 1. 기술 스택
- 프론트엔드: HTML/CSS/JavaScript (또는 React, Vue 등)
- 백엔드: 필요 없음 (또는 Node.js, Python 등)
- 데이터베이스: 필요 없음 (또는 Firebase, Supabase 등)
- 배포: GitHub Pages (또는 Vercel, Netlify 등)

## 2. 폴더 구조
```
프로젝트명/
├── index.html      # 메인 페이지
├── style.css       # 스타일
├── script.js       # 기능
└── assets/         # 이미지, 폰트 등
```

## 3. 화면 구성 (와이어프레임)
```
┌─────────────────────────┐
│        제목             │
├─────────────────────────┤
│   [문장 표시 영역]       │
├─────────────────────────┤
│   [입력창]              │
├─────────────────────────┤
│  [버튼1]  [버튼2]       │
└─────────────────────────┘
```

## 4. 데이터 구조
- 문장 목록: 배열로 저장
- 점수: 변수로 계산
- 저장: localStorage (브라우저 저장)
```

## 4️⃣ 체크리스트 (Tasks)

### 뭔가요?
**"할 일 목록"** 하나씩 체크하며 진행

### 체크리스트 작성법

**큰 기능 → 작은 태스크로 분해**

```markdown
# 체크리스트

## UI 만들기
- [ ] HTML 기본 구조 작성
- [ ] 제목 영역 만들기
- [ ] 문장 표시 영역 만들기
- [ ] 입력창 만들기
- [ ] 버튼 만들기
- [ ] CSS 스타일 적용
- [ ] 반응형 (모바일) 적용

## 기능 구현
- [ ] 문장 데이터 준비
- [ ] 시작 버튼 기능
- [ ] 입력 체크 기능
- [ ] 정확도 계산
- [ ] 타수 계산
- [ ] 다음 문장 넘기기
- [ ] 완료 메시지

## 테스트 & 배포
- [ ] PC에서 테스트
- [ ] 모바일에서 테스트
- [ ] 버그 수정
- [ ] GitHub 업로드
- [ ] 배포 링크 확인
```

## 5️⃣ 개발 (Coding)

### 핵심 원칙

```
🎯 MVP 먼저! (Minimum Viable Product)
   = 최소한의 기능으로 먼저 동작하게 만들기
   = 예쁘게 꾸미는 건 나중에!

🎯 작게 나눠서!
   = 한 번에 다 만들려고 하지 말기
   = 기능 하나씩 추가

🎯 자주 테스트!
   = 조금 만들고 → 확인
   = 문제 있으면 바로 수정
```

### AI와 함께 개발하기

```
1. PRD 보여주기
   "이 문서대로 만들어줘"

2. 한 기능씩 요청
   "먼저 UI만 만들어줘"
   "이제 시작 버튼 기능 추가해줘"

3. 에러 나면 복붙
   "이런 에러가 나와: [에러 메시지]"
   
4. 수정 요청
   "버튼을 더 크게 해줘"
   "색상을 주황색으로 바꿔줘"
```

## 7️⃣ 배포 (Deploy)

### 무료 배포 방법

| 서비스 | 특징 | 난이도 |
|--------|------|--------|
| **GitHub Pages** | 무료, 간단 | ⭐ 쉬움 |
| **Vercel** | 무료, 빠름 | ⭐⭐ 보통 |
| **Netlify** | 무료, 편리 | ⭐⭐ 보통 |

### GitHub Pages 배포 방법

```
1. GitHub 가입
2. 새 저장소(Repository) 만들기
3. 파일 업로드
4. Settings > Pages > 활성화
5. 링크 받기! 🎉
```

## 8️⃣ 피드백 & 개선 (Iterate)

### 피드백 받는 방법

```
1. 직접 사용해보기
   - "불편한 점이 뭐지?"
   
2. 다른 사람한테 써보게 하기
   - "어디서 막혔어요?"
   
3. 댓글/리뷰 확인
   - "이 기능 추가해주세요"
```

### 개선 사이클

```
사용 → 불편 발견 → 개선 → 다시 사용 → ...
        (무한 반복!)
```
