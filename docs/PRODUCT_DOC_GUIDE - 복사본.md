# 프로덕트 문서화 가이드

## 문서 작성 순서

```
1️⃣ PRD → 2️⃣ 기술문서 → 3️⃣ 로드맵 → 4️⃣ 체크리스트
```

| 순서 | 문서 | 핵심 질문 | 작성 시점 |
|------|------|----------|----------|
| 1️⃣ | PRD | "무엇을 만들까?" | 기획 단계 |
| 2️⃣ | TECH | "어떻게 만들까?" | 설계 단계 |
| 3️⃣ | ROADMAP | "언제까지 만들까?" | 계획 단계 |
| 4️⃣ | CHECKLIST | "뭘 했고 뭐가 남았나?" | 개발 단계 |

---

## 파일명 규칙

```
{순번}_{문서유형}_{날짜}_{제품명}.md
```

**예시:**
```
products/ghost-prompt/docs/
├── 01_PRD_2026-02-02_고스트프롬프트.md
├── 02_TECH_2026-02-02_고스트프롬프트.md
├── 03_ROADMAP_2026-02-02_고스트프롬프트.md
└── 04_CHECKLIST_2026-02-02_고스트프롬프트.md
```

---

## 1️⃣ PRD (Product Requirements Document)

### 목적
제품이 **무엇**인지, **왜** 만드는지, **누구**를 위한 것인지 정의

### 필수 섹션
```markdown
# {제품명} PRD

## 1. 제품 개요
- 제품명
- 한 줄 소개
- 타겟 사용자
- 핵심 문제 & 솔루션

## 2. 기능 요구사항
- 핵심 기능 (MVP)
- 추가 기능 (v2)

## 3. UI/UX 설계
- 화면 구성
- 색상/폰트

## 4. 시스템 요구사항
- OS, 메모리 등

## 5. 성공 지표
- KPI, 목표
```

---

## 2️⃣ TECH (기술 설계서)

### 목적
제품을 **어떻게** 만들지 기술적으로 설계

### 필수 섹션
```markdown
# {제품명} 기술 설계서

## 1. 아키텍처 개요
- 시스템 구조도
- 모듈 관계

## 2. 기술 스택
- 언어, 프레임워크, 라이브러리

## 3. 핵심 기술
- 주요 알고리즘/API 설명
- 코드 예시

## 4. 모듈/클래스 설계
- 주요 클래스 구조
- 메서드 설명

## 5. 데이터 구조
- 설정 파일 형식
- 저장소 구조

## 6. 에러 처리
- 예외 상황 대응

## 7. 테스트 계획
- 테스트 항목
```

---

## 3️⃣ ROADMAP (로드맵)

### 목적
**언제까지** 무엇을 완성할지 일정 계획

### 필수 섹션
```markdown
# {제품명} 로드맵

## Phase 1: MVP
- 목표
- 기능 목록
- 예상 기간

## Phase 2: 개선
- 목표
- 기능 목록
- 예상 기간

## Phase 3: 배포
- 목표
- 작업 목록

## 마일스톤 요약
| 버전 | 목표 | 예상 완료 |
|------|------|----------|
| v0.1 | MVP | Day 1 |
| v1.0 | 정식 | Day 5 |
```

---

## 4️⃣ CHECKLIST (체크리스트)

### 목적
실제 개발 진행 상황 추적

### 필수 섹션
```markdown
# {제품명} 개발 체크리스트

## 현재 상태
- 🟢 완료: X개
- 🟡 진행중: X개
- ⚪ 대기: X개

## Phase 1: MVP
- [x] 완료된 태스크
- [ ] 미완료 태스크

## Phase 2: 개선
- [ ] 태스크 1
- [ ] 태스크 2

## 이슈/메모
- 발생한 문제와 해결책
```

## 폴더 구조

```
products/{제품명}/
├── README.md                    # 제품 소개 (간단)
├── {제품}.py                    # 메인 코드
├── ui-mockup.html               # UI 목업 (선택)
└── docs/
    ├── 01_PRD_{날짜}_{제품명}.md
    ├── 02_TECH_{날짜}_{제품명}.md
    ├── 03_ROADMAP_{날짜}_{제품명}.md
    └── 04_CHECKLIST_{날짜}_{제품명}.md
```

---

## 작성 팁

1. **PRD 먼저**: 뭘 만들지 모르면 설계 불가
2. **기술문서 다음**: 어떻게 만들지 알아야 일정 산정
3. **로드맵은 유연하게**: 일정은 바뀔 수 있음
4. **체크리스트는 매일 업데이트**: 진행 상황 추적

---

## 날짜 형식

- **YYYY-MM-DD** 사용 (예: 2026-02-02)
- 문서 최초 작성일 기준
- 수정 시 문서 내 "최종 수정일" 추가
