## 반복 업무에 지치셨나요?

"매일 똑같은 이메일 복사해서 복사..."  
"폼 응답을 하나하나 엑셀에 옮기는 게 너무 tedious해요"  
"SNS에 올린 글을 여러 플랫폼에 일일이 올려야 해요"

이런 일들, AI가 대신해줄 수 있어요!  
**Zapier와 Make**라는 도구를 사용하면, 마치 레고 블록을 조립하듯이 업무를 자동화할 수 있습니다.

## 1. 업무 자동화가 뭔가요?

### 1-1. 쉽게 이해하기

**정의:**  
컴퓨터가 사람 대신 **규칙적인 작업**을 자동으로 수행하도록 만드는 것

**쉬운 비유:**
> 예전에는 편지를 쓰면 집배원이 직접 배달했어요.  
> 지금은 이메일이 자동으로 전송되죠?  
> 업무 자동화도 똑같아요.  
> "A가 일어나면 B를 합니다"라는 규칙을 정핆두면  
> 컴퓨터가 24시간 자동으로 실행해줘요.

### 1-2. 자동화하면 좋은 점

**시간 절약**
- 하루 1시간씩 절약 → 한 달 22시간!
- 반복 업무에서 핵심 업무로 전환

**실수 방지**
- 사람은 실수하지만 컴퓨터는 안 함
- 데이터 누락 방지

**스트레스 감소**
- 지루한 작업에서 핵방
- 창의적 업무에 집중

**24시간 운영**
- 내가 자도, 주말에도 작동
- 해외와 시차 극복

## 2. Zapier 시작하기

### 2-1. Zapier가 뭔가요?

**특징:**
- **5000+ 앱** 연결 가능
- **코딩 없이** 자동화
- **시각적** 인터페이스
- 웹 기반 (설치 불필요)

**가격:**
- 묶: 월 100작업 (Zap 5개)
- Starter: $19.99/월 (750작업)
- Professional: $49/월 (2,000작업)

### 2-2. 기본 개념: Trigger와 Action

**Trigger (트리거):**
```
"무엇이 일어나면" - 자동화의 시작점

예시:
- Gmail에서 새 이메일 수신
- 구글 폼에 새 응답 제출
- Slack에서 특정 키워드 언급
- 매일 오전 9시
```

**Action (액션):**
```
"무엇을 할까요" - 자동화가 수행할 작업

예시:
- 이메일 복사해서 스프레드시트에 추가
- 슬랙으로 알림 볂기
- CRM에 리드 생성
- 파일을 드롭박스에 저장
```

**Zap (자동화 시나리오):**
```
Trigger + Action = Zap

예: 
[구글 폼 응답] → [스프레드시트 기록] → [슬랙 알림]
```

### 2-3. 첫 Zap 만들어보기

**시나리오:**  
"구글 폼 응답이 들어오면 스프레드시트에 자동 기록"

**만드는 법:**

**Step 1: Zapier 가입**
```
1. zapier.com 접속
2. 회원가입 (구글 계정으로 가능)
3. 대시보드 진입
```

**Step 2: Zap 생성**
```
1. "Create Zap" 버튼 클릭
2. 이름 입력: "구글폼 응답 자동화"
```

**Step 3: Trigger 설정**
```
App: Google Forms
Event: New Response in Spreadsheet

설정:
- 스프레드시트 선택
- 워크시트 선택

Test:
- "Test trigger" 클릭
- 최근 응답 데이터 확인
```

**Step 4: Action 설정**
```
App: Google Sheets
Event: Create Spreadsheet Row

설정:
- Drive: My Google Drive
- Spreadsheet: 원하는 시트 선택
- Worksheet: 시트 선택

Map fields:
- Timestamp → Timestamp 열
- Email → Email 열
- Name → Name 열
- Message → Message 열

Test:
- "Test & Continue" 클릭
- 스프레드시트 확인
```

**Step 5: Zap 켜기**
```
"Publish" 버튼 클릭
→ Zap이 활성화됨!
```

## 3. Make (Integromat) 시작하기

### 3-1. Make가 뭔가요?

**특징:**
- **시각적** 플로우 디자인
- **복잡한 로직** 가능 (if/else, 반복 등)
- **데이터 변환** 강력
- 더 많은 **묶 기능**

**Zapier vs Make:**
| 특징 | Zapier | Make |
|------|--------|------|
| 사용성 | 쉬움 | 중간 |
| 복잡한 로직 | 제한적 | 강력 |
| 묶 사용량 | 적음 | 많음 |
| 데이터 변환 | 기본 | 고급 |
| 시각화 | 리스트 | 다이어그램 |

### 3-2. 시나리오(Scenario) 만들기

**시나리오:**  
"이메일에 첨부파일이 오면 드롭박스에 저장 + 슬랙 알림"

**만드는 법:**

**Step 1: Make 가입**
```
make.com 접속 → 회원가입
```

**Step 2: Scenario 생성**
```
"Create a new scenario" 클릭
```

**Step 3: 모듈 추가 (시각적 설계)**
```
1. Gmail 모듈 추가
   - Watch emails
   - 필터: has attachment

2. Dropbox 모듈 추가
   - Upload a file
   - 폰더 경로 설정

3. Slack 모듈 추가
   - Send a message
   - 채널/메시지 설정
```

**Step 4: 연결 설정**
```
각 모듈을 선으로 연결
→ 데이터 흐름 시각화
```

**Step 5: 실행 테스트**
```
"Run once" 클릭
→ 테스트 이메일 볂기
→ 결과 확인
```

**Step 6: 자동화 활성화**
```
"Scheduling" ON
→ 주기 설정 (즉시, 15분, 1시간 등)
```

## 4. 실전 자동화 시나리오 10선

### 시나리오 1: 리드 관리 자동화

**상황:**
```
웹사이트 문의 → 메일 확인 → 엑셀 입력 → CRM 등록 → 담당자 배정
(수동으로 하면 15분 소요)
```

**자동화:**
```
[웹사이트 문의폼]
    ↓
[이메일 파싱] (Zapier)
    ↓
[구글 시트 기록]
    ↓
[HubSpot CRM 자동 등록]
    ↓
[영업팀 슬랙 알림]
    ↓
[담당자 자동 배정 (라운드 로빈)]
```

**효과:**  
즉시 처리, 리드 누락 0%

### 시나리오 2: SNS 콘텐츠 자동 배포

**상황:**
```
인스타그램에 올린 글을 트위터, 페이스북, 링크드인에도 올려야 함
```

**자동화:**
```
[인스타그램 게시물]
    ↓
[이미지 다운로드]
    ↓
[텍스트 변환 (해시태그 제거)]
    ↓
[트위터: 텍스트 + 링크]
[페이스북: 전체 내용]
[링크드인: 비즈니스 톤으로 변환]
```

**주의:**  
각 플랫폼 특성에 맞게 변환 필요

### 시나리오 3: 회의록 자동화

**상황:**
```
줌 회의 → 녹화 → 텍스트 변환 → 요약 → 공유
```

**자동화:**
```
[줌 회의 종료]
    ↓
[녹화 파일 클라우드 저장]
    ↓

    ↓

    ↓

    ↓
[참석자들에게 메일 발송]
```

### 시나리오 4: 재고 관리 알림

**상황:**
```
재고가 10개 이하로 떨어지면 발주 필요
```

**자동화:**
```
[구글 시트: 재고 현황]
    ↓ (매일 오전 9시 체크)
[조건: 재고 < 10]
    ↓
[YES]
    ↓
[발주 담당자에게 이메일]
[구매팀 슬랙 채널 알림]
    ↓
[발주서 드래프트 생성]

[NO]
    ↓

```

## 5. 고급 팁과 주의사항

### 5-1. 자동화할 때 주의할 점

**⚠️ 주의사항:**

**1) 테스트 충분히 하기**
```
- 실제 데이터로 테스트
- 예외 상황 테스트
- 에러 처리 확인
```

**2) 에러 핸들링**
```
- 실패 시 알림 설정
- 예비 경로 마련
- 로그 기록 확인
```

**3) 과도한 자동화 주의**
```
- 사람의 판단이 필요한 것은 자동화 금지
- 민감한 결제/승인은 수동으로
- 고객 대응은 자동화에 한계
```

**4) 보안**
```
- 중요 계정 정보는 안전하게
- API 키 정기 교체
- 접근 권한 최소화
```

### 5-2. 문제 해결 가이드

**Zap가 작동 안 할 때:**
```
1. Zapier 대시보드에서 "Zap History" 확인
2. 에러 메시지 확인
3. 각 단계별로 테스트 재실행
4. 연결된 계정 권한 확인
5. 데이터 형식 확인 (날짜, 숫자 등)
```

**Make 시나리오 오류:**
```
1. 실행 로그 확인
2. 각 모듈의 입력/출력 데이터 확인
3. 데이터 매핑 확인
4. 필터 조건 확인
5. API 제한 확인 (호출 횟수)
```

## 6. 나만의 자동화 설계하기

### 설계 프로세스

**Step 1: 현재 업무 분석**
```
"어떤 반복 업무를 매일/매주 하고 있나?"

체크리스트:
□ 매일 하는 데이터 입력
□ 동일한 형식의 이메일 발송
□ 여러 플랫폼에 동일 내용 게시
□ 정기적인 데이터 백업
□ 리드/문의 처리
□ 보고서 작성
□ 승인 프로세스
```

**Step 2: 자동화 후보 선정**
```
우선순위 매트릭스:

              자주함
                 │
    높은 우선순위 │   후보 1 (예: 리드 입력)
    (즉시 자동화)│
                 │
  안 자주함 ─────┼────────── 자주함
                 │
                 │   후보 4 (예: 연말 정산)
    낮은 우선순위│   (예산 허락 시)
                 │
              안 자주함
```

**Step 3: 시나리오 설계**
```
템플릿:

[자동화 이름]
목적: [왜 자동화하는지]
빈도: [얼마나 자주]
Trigger: [시작점]
Action 1: [첫 번째 작업]
Action 2: [두 번째 작업]
...
결과: [최종 결과물]

예상 효과:
- 시간 절약: [시간/회]
- 오류 감소: [%]
```

**Step 4: 구현 및 테스트**
```
1. 간단한 버전부터 시작
2. 하나씩 기능 추가
3. 충분히 테스트
4. 팀원 교육
5. 문서화
```
