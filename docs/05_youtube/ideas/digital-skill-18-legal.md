## 1️⃣ Claude/GPT-4 - AI 계약서 분석

### AI 계약서 분석이란?

**Claude**나 **GPT-4** 같은 LLM을 활용해 계약서를 **요약하고 위험 요소를 파악**하는 방법입니다. 별도 도구 없이도 강력한 법률 어시스턴트 역할을 합니다.

### 핵심 기능

**1. 계약서 요약**
- 핵심 조항 추출
- 권리/의무 정리
- 날짜 및 금액 파악

**2. 위험 요소 식별**
- 불리한 조항 감지
- 모호한 표현 경고
- 법적 리스크 분석

**3. 비교 분석**
- 이전 버전과 비교
- 표준 계약과 차이점
- 삭제/추가된 내용 파악

**4. 질문 답변**
- 계약 내용 질의응답
- 특정 조항 설명
- 대안 제시

### 실습: NDA 검토하기

**시나리오**: 외주 개발자와의 NDA(비밀유지계약) 검토

**Step 1**: 계약서 업로드
```
Claude/GPT-4에 입력:

"다음 NDA 계약서를 검토해주세요.
주요 조항을 요약하고, 
잠재적 위험 요소를 지적해주세요.

[NDA 전문 붙여넣기]"
```

**Step 2**: AI 분석 결과
```
Claude의 분석 예시:

📋 계약 요약:
─────────────────────
당사자:
- 의뢰인: [회사명] (이하 "갑")
- 수행자: [외주 개발자] (이하 "을")

주요 내용:
1. 비밀정보 범위: 기술 문서, 소스코드, 사업 계획
2. 유지 기간: 계약 종료 후 3년
3. 위약금: ₩50,000,000
4. 준거법: 대한민국 법

⚠️ 주의사항:
─────────────────────
1. [위험] 비밀정보 정의가 모호함
   → "기타 갑이 지정하는 정보" 
   → 구체적 나열 권장

2. [주의] 위약금이 과도하게 높음
   → 실제 손해액 입증 필요
   → 법원에서 감액 가능성

3. [개선] 해지 조항 미비
   → 양측 해지권 명시 필요
   → 데이터 반환 절차 추가 권장

✅ 양호한 점:
─────────────────────
- 준거법 및 관할법원 명확
- 지식재산권 귀속 조항 명확
- 분쟁 해결 절차 구체적
```

**Step 3**: 개선 요청
```
추가 프롬프트:

"위 NDA의 문제점을 개선한 
개정안을 작성해주세요.

특히 다음 사항을 반영해주세요:
1. 비밀정보 정의 구체화
2. 위약금 현실적 조정
3. 해지 조항 추가
4. 데이터 삭제 절차 명시"
```

### 고급 프롬프트 템플릿

```
계약서 검토 마스터 프롬프트:

"당신은 10년 경력의 기업 변호사입니다.
다음 계약서를 검토하고 제공된 형식으로 
분석해주세요.

계약서: [계약서 전문]

분석 형식:
1. 계약 개요 (3줄 요약)
2. 주요 조항 목록
3. 위험 요소 (높음/중간/낮음)
4. 권장 개선사항
5. 협상 포인트
6. 결론 (체결 권고 여부)

주의: 법적 자문이 아닌 참고용으로만 
작성해주세요."
```

## 2️⃣ LawGeex - 기업용 AI 계약서 검토

### LawGeex란?

**LawGeex**는 AI가 계약서를 **자동으로 검토하고 승인/수정**을 권장하는 엔터프라이즈 도구입니다. 대용량 계약 처리에 최적화되어 있습니다.

### 핵심 기능

**1. 자동 검토**
- 100+ 조항 자동 분석
- 회사 정책과 비교
- 위험도 자동 산정

**2. 플레이북 기반 검토**
- 사전 설정된 승인 기준
- 부서별/계약별 차등 적용
- 자동 승인 워크플로우

**3. 리포팅 및 분석**
- 계약 현황 대시보드
- 병목 구간 식별
- 평균 검토 시간 추적

### 실습: 공급계약 자동 검토

**시나리오**: 50개 공급업체 계약서 자동 검토

**Step 1**: 플레이북 설정
```
Company Playbook:

계약 유형: 공급계약

필수 포함 조항:
☑ 납기 위약금 (일 0.5% 이상)
☑ 품질 보증 기간 (1년 이상)
☑ 지식재산 귀속 (구매사)
☑ 비밀유지 (3년 이상)
☑ 해지 사유 명시

금지 조항:
✗ 무제한 책임
✗ 자동 갱신
✗ 단독 해지권 (공급사)
✗ 준거법 외국법

경고 조항 (검토 필요):
△ 제3자 하도급 허용
△ 가격 조정 조항
△ 최소 구매 수량
```

**Step 2**: AI 자동 검토
```
검토 결과 예시:

계약서 #23 - 공급사 A
━━━━━━━━━━━━━━━━━━━━
전체 위험도: MEDIUM
예상 검토 시간: 15분

✅ 통과 (자동 승인 가능):
- 납기 위약금: 0.8%/일 ✓
- 품질 보증: 2년 ✓
- 지식재산: 구매사 귀속 ✓

⚠️ 검토 필요:
- 비밀유지: 2년 (3년 미만)
- 준거법: 싱가포르법 (변경 필요)

❌ 위반:
- 자동 갱신 조항 발견
- 수정 권장

권장 조치: 수동 검토 후 협상
```

## 3️⃣ LegalZoom - 스타트업 법률 서비스

### LegalZoom란?

**LegalZoom**은 **스타트업과 소기업**을 위한 종합 법률 서비스 플랫폼입니다. 표준 계약서 템플릿과 변호사 매칭을 제공합니다.

### 핵심 기능

**1. 계약서 템플릿**
- 100+ 표준 계약서
- 업종별 맞춤 템플릿
- 한국어 번역본 제공

**2. 전자 서명**
- DocuSign 통합
- 법적 효력 보장
- 추적 및 관리

**3. 변호사 매칭**
- 전문 분야별 변호사 연결
- 고정 비용 상담
- 온디맨드 법률 자문

### 실습: 스타트업 필수 계약서 세팅

**Step 1**: 필수 계약서 목록
```
스타트업 필수 계약 10종:

1. 정관 (Articles of Incorporation)
2. 주주간 계약서 (Shareholders Agreement)
3. 근로계약서 (Employment Contract)
4. NDA (Non-Disclosure Agreement)
5. 지식재산 귀속 계약서 (IP Assignment)
6. 외주 개발 계약서
7. SaaS 이용약관 (Terms of Service)
8. 개인정보처리방침 (Privacy Policy)
9. 투자 계약서 (Term Sheet)
10. 해지/퇴사 합의서
```

**Step 2**: 근로계약서 템플릿 커스터마이징
```
LegalZoom 템플릿 수정:

기본 정보:
□ 회사명: [스타트업명]
□ 주소: 서울시 강남구
□ 사업자번호: [번호]

근로 조건:
□ 직책: 개발자
□ 급여: 월 ₩4,000,000
□ 근무지: 서울 본사 (주 3회 출근)
□ 근무시간: 유연근무제

특약사항:
□ probation 기간: 3개월 (급여 90%)
□ 주식옵션: 4년 상환, 1년 cliff
□ 비밀유지: 퇴사 후 2년
□ 경쟁금지: 퇴사 후 1년 (주주인 경우)

지식재산:
□ 근무 중 개발한 모든 IP는 회사 귀속
□ 사전 동의 없이 오픈소스 공개 금지
```

**1. 전자 서명**
- 법적 효력 완벽 보장
- 180+ 국가 사용 가능
- 모바일 최적화

**2. AI 계약 분석 (DocuSign AI)**
- 계약서 자동 분류
- 핵심 조항 추출
- 위험 요소 경고

**3. 워크플로우 자동화**
- 승인 체인 설정
- 알림 및 리마인더

## 🎯 계약서 관리 전략

### 계약서 라이프사이클 관리

```
1. 생성 (Creation)
   ├── 템플릿 선택
   ├── AI 기반 초안 작성
   └── 법무 검토 (중요 계약)

2. 검토 (Review)
   ├── AI 자동 검토
   ├── 상대방 검토
   └── 협상 및 수정

3. 체결 (Execution)
   ├── 전자 서명
   ├── 승인 워크플로우
   └── 아카이빙

4. 관리 (Management)
   ├── 만료일 추적
   ├── 갱신 알림
   └── 실적 관리

5. 종료 (Termination)
   ├── 해지 절차
   ├── 데이터 반환
   └── 문서 보관
```

### 계약서 체크리스트 템플릿

```
계약 체결 전 필수 체크:

□ 기본 정보 확인
  - 당사자 명칭/주소 정확성
  - 사업자등록번호 확인
  - 대표자 서명권 확인

□ 핵심 조항 검토
  - 금액/납부 조건
  - 납기/이행 기한
  - 지식재산권 귀속
  - 비밀유지 의무
  - 해지/위약 조항

□ 법적 검토
  - 준거법 및 관할
  - 분쟁 해결 절차
  - 손해배상 한도
  - 보증/면책 조항

□ 실무 검토
  - SLA (서비스 수준)
  - 변경 관리 절차
  - 인수 기준
  - 하자보수 조건

□ 위험 요소
  - 무제한 책임
  - 일방적 해지권
  - 자동 갱신
  - 과도한 위약금
```

## ⚠️ 주의사항 및 한계

### AI 법률 도구의 한계

```
⚠️ 절대 잊지 말아야 할 사실:

1. AI는 법적 자문이 아님
   → 참고용으로만 사용
   → 중요 계약은 변호사 필수

2. 환각(Hallucination) 위험
   → 존재하지 않는 법 조항 인용 가능
   → 반드시 사실 확인 필요

3. 국가별 법률 차이
   → 한국법과 외국법 혼동 가능
   → 현행법 확인 필수

4. 특수한 상황 미흡
   → 표준적인 계약만 유효
   → 복잡한 M&A, 소송 등은 무리
```

### 안전한 사용 원칙

```
✅ 해야 할 것:
- AI 결과를 참고용으로만 사용
- 표준 계약서 작성에 활용
- 변호사 최종 검수
- 정기적인 템플릿 업데이트

❌ 하지 말아야 할 것:
- 중대 계약을 AI만으로 검토
- AI 결과 100% 신뢰
- 법적 분쟁 시 AI 분석 의존
- 저작권 있는 계약서 무단 복사
```
