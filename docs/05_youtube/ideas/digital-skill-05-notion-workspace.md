## 1. Notion이 뭔가요?

### 1-1. 정의와 특징

**Notion = 노트 + 데이터베이스 + 위키 + 프로젝트 관리**

**주요 특징:**
```
✅ 블록 기반: LEGO처럼 조립
✅ 데이터베이스: 스프레드시트처럼
✅ 템플릿: 시작이 쉬움
✅ 협업: 팀과 공유
✅ 크로스 플랫폼: 어디서든
```

**가격:**
```
🆓 Personal (묶):
- 무제한 페이지
- 파일 첨부 (5MB 제한)
- 7일 히스토리

💼 Plus ($8/월):
- 무제한 파일
- 30일 히스토리
- 팀원 100명

🏢 Business ($15/월):
- 팀 기능 강화
- SSO
- 고급 권한
```

### 1-2. 왜 Notion인가?

**기존 방식:**
```
- 메모: 메모앱
- 문서: 워드
- 스프레드시트: 엑셀
- 일정: 구글캘린더
- 파일 저장: 드롭박스
→ 5개 앱을 왔다갔다
```

**Notion 방식:**
```
- 모든 것이 한 곳에
- 서로 연결 가능
- 한 화면에서 전체 파악
- 강력한 검색
```

## 2. 기초 다지기

### 2-1. 페이지 구조 이해하기

**계층 구조:**
```
Workspace (회사/개인)
└── Teamspace (팀별)
    └── Page (페이지)
        ├── Page (하위 페이지)
        └── Database (데이터베이스)
            └── Page (항목)
```

**실제 예시:**
```
🏢 내 워크스페이스
├── 📋 개인
│   ├── 🎯 2024 목표
│   ├── 📚 독서 리스트
│   └── 💪 건강 관리
├── 💼 업무
│   ├── 📊 프로젝트 관리
│   ├── 📅 주간 회의록
│   └── 📖 업무 위키
└── 🏠 가정
    ├── 📝 가계부
    ├── 🍳 레시피
    └── 🎁 선물 리스트
```

### 2-2. 블록(Block) 시스템

**블록이 뭔가요?**
```
Notion의 기본 단위 (레고 블록처럼)
```

**블록 종류:**
```
📝 텍스트:
- Heading 1, 2, 3
- Text
- Bulleted list
- Numbered list
- To-do list

📊 미디어:
- Image
- Video
- File
- Embed (유튜브 등)

🗃️ 데이터:
- Database (Inline/Full page)
- Table
- Board
- Calendar
- Gallery

🔗 기타:
- Link to page
- Page
- Divider
- Quote
- Callout
```

**블록 단축키:**
```
/heading - 제목
/bullet - 글머리
/todo - 체크리스트
/table - 테이블
/page - 페이지
/database - 데이터베이스
```

**페이지 만들기:**
```

3. 제목 입력
4. 아이콘 선택 (선택)
5. 커버 이미지 설정 (선택)
```

**템플릿 사용:**
```
1. 새 페이지
2. "Templates" 클릭
3. 카테고리 선택
4. 원하는 템플릿 선택
5. Duplicate
```

## 3. 데이터베이스(Database) 마스터하기

### 3-1. 데이터베이스가 뭔가요?

**정의:**
```
정보를 체계적으로 저장하고 관리하는 표
엑셀 + 관계형 DB의 장점 결합
```

**뷰(View) 종류:**
```
📊 Table: 표 형태
📋 Board: 칸반 보드
📅 Calendar: 캘린더
🖼️ Gallery: 카드 형태
📄 List: 목록
⏱️ Timeline: 타임라인 (간트차트)
```

### 3-2. 속성(Properties) 종류

**기본 속성:**
```
📝 Text: 텍스트
🔢 Number: 숫자
✅ Select: 단일 선택
☑️ Multi-select: 다중 선택
📅 Date: 날짜
👤 Person: 사람
📁 Files & media: 파일
🔗 URL: 링크
✔️ Checkbox: 체크박스
📧 Email: 이메일
📞 Phone: 전화번호
```

**고급 속성:**
```
🔄 Formula: 계산식
🔗 Relation: 다른 DB와 연결
📝 Rollup: 연결된 DB의 값 가져오기
🆔 Created time: 생성 시간
✏️ Last edited time: 수정 시간
```

### 3-3. 실전 데이터베이스 예시

**예시 1: 업무 관리 시스템**
```
데이터베이스: 업무 목록

속성:
- 작업명 (Title)
- 상태 (Select): 대기/진행중/완료/보류
- 우선순위 (Select): 높음/중간/낮음
- 담당자 (Person)
- 마감일 (Date)
- 프로젝트 (Relation): 프로젝트 DB와 연결
- 완료여부 (Checkbox)

뷰:
- 보드 뷰: 상태별 칸반
- 캘린더 뷰: 마감일 기준
- 테이블 뷰: 전체 목록
- 갤러리 뷰: 프로젝트별
```

**예시 2: 독서 노트**
```
데이터베이스: 독서 리스트

속성:
- 제목 (Title)
- 저자 (Text)
- 상태 (Select): 읽는중/완료/예정
- 평점 (Select): ⭐1-5
- 장르 (Multi-select): 소설/자기계발/경제
- 시작일 (Date)
- 완료일 (Date)
- 소감 (Text)
- 표지 (Files)
```

## 4. Relation과 Rollup 활용

### 4-1. 관계형 데이터베이스

**Relation이 뭔가요?**
```
두 데이터베이스를 연결하는 기능
엑셀의 VLOOKUP과 유사하지만 더 강력
```

**실제 예시:**
```
DB 1: 프로젝트
DB 2: 업무

Relation 설정:
- 업무 DB에 "프로젝트" 속성 추가
- 프로젝트 DB와 연결
→ 각 업무가 어떤 프로젝트에 속하는지 표시
```

### 4-2. Rollup으로 정보 집계

**Rollup이 뭔가요?**
```
연결된 DB의 정보를 가져와 계산
```

**활용 예시:**
```
프로젝트 DB에서:
- "총 업무 수" Rollup
  → 관련 업무 개수 세기

- "완료율" Rollup  
  → 완료된 업무 / 전체 업무
  → Formula: prop("완료") / prop("전체")

- "마감일" Rollup
  → 관련 업무 중 가장 빠른 마감일
```

### 4-3. 실전 시나리오

**시나리오: 콘텐츠 관리 시스템**
```
DB 1: 콘텐츠 아이디어
- 아이디어 제목
- 상태 (아이디어/작성중/완료/발행)
- 작성자
- 카테고리

DB 2: 프로젝트
- 프로젝트명
- 기간
- 상태
- 관련 콘텐츠 (Relation)
- 콘텐츠 수 (Rollup: Count)
- 완료된 콘텐츠 수 (Rollup: Count with filter)
- 진행률 (Formula)
```

## 6. 고급 기능

### 6-1. Formula (공식)

**기초 Formula:**
```
📅 날짜 계산:
dateBetween(end, start, "days")
→ 남은 일수 계산

📊 진행률:
prop("완료") / prop("전체") * 100

📝 문자열:
toUpper(prop("이름"))
→ 대문자 변환
```

**활용 예시:**
```
마감일 D-Day:
"D" + dateBetween(prop("마감일"), now(), "days")
→ D-3, D-1, D+2 등 표시
```

### 6-2. Synced Blocks (동기화 블록)

**기능:**
```
한 곳에서 수정하면 
다른 모든 곳에서 자동 업데이트
```

**활용:**
```
- 팀 규칙/공지
- 자주 변경되는 정보
- 템플릿 내 공통 내용
```

### 6-3. Linked Databases (연결 DB)

**기능:**
```
같은 데이터베이스를 
여러 페이지에서 다른 뷰로 표시
```

**활용:**
```
전체 업무 DB:
- 내 업무 페이지: 필터 (담당자 = 나)
- 마감 임박 페이지: 필터 + 정렬
- 프로젝트별 페이지: 필터 (프로젝트 = A)
```

## 8. 실전 적용 가이드
