## 서론: 데이터 없이는 마케팅이 불가능하다

디지털 마케팅의 핵심은 '측정'입니다. 어떤 채널이 효과적인지, 어떤 캠페인이 매출을 가져오는지, 고객이 웹사이트에서 어떤 행동을 하는지 알지 못하면 전략적 의사결정이 불가능합니다.

구글 애널리틱스 4(GA4)는 웹사이트와 앱의 사용자 행동을 추적하고 분석하는 가장 강력한 무료 도구입니다. 본 가이드에서는 GA4를 처음부터 올바르게 설정하는 방법을 단계별로 설명합니다.

## 1. GA4 vs Universal Analytics: 왜 바꿔야 하나?

### 주요 차이점

| 항목 | Universal Analytics | GA4 |
|------|-------------------|-----|
| **데이터 모델** | 세션 기반 | 이벤트 기반 |
| **측정 방식** | 페이지뷰 중심 | 모든 상호작용 측정 |
| **크로스 디바이스** | 제한적 | 통합 추적 |
| **머신러닝** | 없음 | 내장 예측 기능 |
| **보고서** | 사전 정의 | 탐색적 분석 중심 |

### GA4의 장점

**1) 통합 데이터 모델**
- 웹과 앱 데이터를 하나로 통합
- 사용자 여정 전체 파악 가능
- 크로스 플랫폼 분석 용이

**2) 향상된 추적 기능**
- 자동 이벤트 수집 (스크롤, 클릭, 동영상 등)
- 향상된 측정 기능으로 별도 코드 없이 추적
- 머신러닝 기반 예측 지표

**3) 미래 지향적**
- 쿠키 없는 세계 대응
- 개인정보 보호 규정 준수
- 지속적인 업데이트

## 2. GA4 계정 및 속성 생성

### 2.1 계정 생성

**1) Google Analytics 접속**
- https://analytics.google.com
- Google 계정으로 로그인

**2) 관리자 설정**
```
왼쪽 하단 톱니바퀴 아이콘 클릭
→ 계정 만들기
→ 계정 이름 입력 (예: "부자타임")
→ "다음" 클릭
```

**계정 설정 옵션:**
- **계정 데이터 공유 설정**: Google 제품 및 서비스 연결 권장
- **비즈니스 정보**: 산업 카테고리, 비즈니스 규모 선택

### 2.2 속성(Property) 생성

**속성 설정:**
```
속성 이름: "부자타임 공식 웹사이트"
보고 시간대: (UTC+09:00) 서울
통화: 대한민국 원(KRW)
```

**비즈니스 정보 입력:**
- 산업 카테고리: "제조업" 또는 "광고/마케팅"
- 비즈니스 규모: 직원 수 선택
- GA4 도구 사용 목적 체크

### 2.3 데이터 스트림 설정

**웹 스트림 생성:**
```
1. "데이터 수집 시작" 클릭
2. "웹" 선택
3. 웹사이트 URL 입력: https://www.example.com
4. 스트림 이름: "메인 웹사이트"
5. "스트림 만들기" 클릭
```

**측정 ID 확인:**
- 생성 후 제공되는 "G-XXXXXXXXXX" 형식의 ID 기록
- 이 ID로 웹사이트와 GA4 연결

## 3. GA4 추적 코드 설치

### 3.1 직접 설치 (gtag.js)

**HTML head 태그에 추가:**

```html
<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());

  gtag('config', 'G-XXXXXXXXXX');
</script>
```

**설치 위치:**
- 모든 페이지의 `<head>` 태그 내
- CMS의 경우 테마 헤더 또는 전역 헤더 설정

### 3.2 Google Tag Manager (GTM) 설치 (권장)

**GTM 계정 생성:**
1. https://tagmanager.google.com 접속
2. 계정 및 컨테이너 생성
3. GTM 코드 설치 (head + body)

**GA4 태그 설정:**
```
1. GTM → 태그 → "새로 만들기"
2. 태그 이름: "GA4 - 메인"
3. 태그 유형: "Google 애널리틱스: GA4 설정"
4. 측정 ID: G-XXXXXXXXXX 입력
5. 트리거: "All Pages" 선택
6. 저장 및 제출
```

### 3.3 CMS별 설치 방법

**WordPress:**
- 플러그인: "Site Kit by Google" 또는 "GA Google Analytics"
- 설정 → 측정 ID 입력

**Wix:**
- 설정 → 마케팅 및 SEO → Analytics
- GA4 ID 입력

**Shopify:**
- 온라인 스토어 → 기본 설정
- Google Analytics 계정 섹션에 ID 입력

### 3.4 설치 확인

**디버그 모드 사용:**
1. GA4 → 관리자 → 디버그 뷰
2. 웹사이트 방문
3. 실시간 데이터 확인

**Tag Assistant (Chrome 확장):**
- Chrome 웹스토어에서 설치
- 웹사이트에서 실행하여 태그 설치 확인

## 4. 핵심 설정: 향상된 측정 및 이벤트

### 4.1 향상된 측정 활성화

**자동 이벤트 설정:**
```
관리자 → 데이터 스트림 → 웹 스트림 선택
→ 향상된 측정 → 톱니바퀴 아이콘
→ 모두 켜기
```

**수집되는 자동 이벤트:**
- **페이지 조회**: 모든 페이지 방문
- **스크롤**: 90% 스크롤 도달
- **클릭**: 외부 링크 클릭
- **양식 상호작용**: 폼 시작, 제출
- **동영상 참여**: 재생, 완료, 진행률
- **파일 다운로드**: PDF 등 다운로드
- **사이트 검색**: 내부 검색어 추적

### 4.2 맞춤 이벤트 설정

**중요 이벤트 전환 설정:**

**1) 연락처 폼 제출**
```javascript
gtag('event', 'contact_form_submit', {
  'event_category': 'form',
  'event_label': 'contact_page',
  'value': 1
});
```

**2) 전화번호 클릭**
```javascript
gtag('event', 'phone_click', {
  'event_category': 'contact',
  'event_label': 'header_phone'
});
```

**3) 이메일 클릭**
```javascript
gtag('event', 'email_click', {
  'event_category': 'contact',
  'event_label': 'footer_email'
});
```

**4) 파일 다운로드**
```javascript
gtag('event', 'file_download', {
  'event_category': 'content',
  'event_label': 'whitepaper_2024',
  'file_name': 'marketing-guide.pdf',
  'file_extension': 'pdf'
});
```

**5) 버튼 클릭**
```javascript
gtag('event', 'cta_click', {
  'event_category': 'engagement',
  'event_label': 'hero_banner',
  'button_text': '무료 상담 받기'
});
```

### 4.3 전환 이벤트 설정

**주요 전환으로 표시:**
```
관리자 → 이벤트 → "모든 이벤트"
→ 전환으로 표시할 이벤트 토글 ON
→ 체크 표시 확인
```

**권장 전환 이벤트:**
- 폼 제출 (form_submit)
- 전화번호 클릭 (phone_click)
- 이메일 클릭 (email_click)
- 구매 완료 (purchase)
- 견적 요청 (quote_request)
- 뉴스레터 구독 (newsletter_subscribe)

## 5. 핵심 보고서 설정 및 활용

### 5.1 실시간 보고서

**활용 방법:**
```
홈 → 실시간
→ 현재 웹사이트 사용자 수 확인
→ 인기 있는 페이지 확인
→ 이벤트 카운트 실시간 확인
```

**체크 포인트:**
- 캠페인 런칭 후 실시간 유입 확인
- 이벤트 트래킹 정상 작동 여부
- 특정 페이지 트래픽 모니터링

### 5.2 사용자 특성 보고서

**주요 지표:**
- **신규 vs 재방문**: 사용자 충성도
- **인구 통계**: 연령, 성별, 관심사
- **지리**: 국가, 도시별 방문
- **기술**: 사용 기기, 브라우저, OS

**활용:**
- 타겟 오디언스 검증
- 광고 타겟팅 최적화
- 콘텐츠 전략 수립

### 5.3 획득 보고서

**채널별 분석:**
```
라이프사이클 → 획득 → 트래픽 획득
→ 세션 기본 채널 그룹별 확인
→ 사용자, 세션, 참여율 비교
```

**주요 채널:**
- **Organic Search**: SEO 트래픽
- **Paid Search**: 광고 트래픽
- **Direct**: 직접 방문
- **Referral**: 외부 사이트 유입
- **Social**: 소셜 미디어
- **Email**: 이메일 마케팅
- **Organic Social**: 유기적 소셜

**분석 포인트:**
- 어떤 채널에서 가장 많은 전환이 발생하는가?
- 각 채널의 참여도(체류 시간, 페이지/세션)는?
- 전환율이 가장 높은 채널은?

### 5.4 참여 보고서

**페이지 및 화면:**
```
라이프사이클 → 참여 → 페이지 및 화면
→ 조회수 상위 페이지 확인
→ 평균 참여 시간 분석
```

**이벤트:**
```
라이프사이클 → 참여 → 이벤트
→ 발생한 모든 이벤트 확인
→ 이벤트별 카운트 및 매개변수
```

**분석 포인트:**
- 인기 페이지와 이탈 페이지
- 이벤트 발생 빈도
- 사용자 여정의 병목 지점

### 5.5 수익 창출 보고서

**전자상거래 추적:**
```javascript
gtag('event', 'purchase', {
  'transaction_id': 'T12345',
  'value': 250000,
  'currency': 'KRW',
  'items': [{
    'item_id': 'SKU123',
    'item_name': '프리미엄 패키지',
    'price': 250000,
    'quantity': 1
  }]
});
```

**전환 경로:**
```
광고 → 수익 창출 → 전환 경로
→ 사용자의 전환까지 여정 확인
→ 각 접점별 기여도 분석
```

## 6. 고급 설정: 맞춤 측정항목 및 차원

### 6.1 맞춤 측정항목 (Custom Metrics)

**생성 방법:**
```
관리자 → 속성 열 → 맞춤 정의 → 맞춤 측정항목
→ "맞춤 측정항목 만들기"
→ 이름, 범위, 설명 입력
→ 이벤트 매개변수 연결
```

**활용 예시:**
- **평균 주문 가치**: 매출 / 거래
- **리드당 비용**: 광고비 / 리드 수
- **페이지당 가치**: 수익 / 페이지뷰

### 6.2 맞춤 차원 (Custom Dimensions)

**사용자 특성:**
- 회원 등급 (VIP, 일반)
- 고객 유형 (B2B, B2C)
- 가입 경로

**설정:**
```javascript
gtag('config', 'G-XXXXXXXXXX', {
  'custom_map': {
    'custom_parameter_1': 'user_type'
  }
});

gtag('event', 'page_view', {
  'user_type': 'B2B'
});
```

### 6.3 맞춤 보고서 생성

**탐색 보고서:**
```
왼쪽 메뉴 → 탐색
→ "비어 있는 탐색" 또는 템플릿 선택
→ 변수(측정항목, 차원) 추가
→ 탭(자유형식, 폭포, 퍼널 등) 설정
```

**유용한 맞춤 보고서:**
1. **리드 생성 보고서**: 폼 제출 추적
2. **콘텐츠 성과**: 블로그 포스트별 성과
3. **캠페인 ROI**: 마케팅 채널별 수익성
4. **사용자 여정**: 퍼널별 전환율

## 7. Google Ads 연동

### 7.1 연결 설정

**연결 방법:**
```
관리자 → 제품 연결 → Google Ads 연결
→ Google Ads 계정 선택
→ "연결" 클릭
```

**자동 태깅 활성화:**
- Google Ads 계정 설정
- 추적 → 자동 태깅 켜기
- GA4에서 광고 성과 자동 추적

### 7.2 잠재고객 연동

**GA4 잠재고객 Google Ads로 공유:**
```
홍보 → 잠재고객
→ 공유할 잠재고객 선택
→ "Google Ads에 게시" 클릭
```

**활용:**
- 웹사이트 방문자 리타겟팅
- 유사 잠재고객 (Lookalike) 생성
- 전환 완료자 제외

## 8. 데이터 정확성 확보

### 8.1 내부 트래픽 필터링

**설정 방법:**
```
관리자 → 데이터 스트림 → 웹 스트림
→ "더보기 태그 설정" → 내부 트래픽 정의
→ "내부 트래픽 만들기"
→ IP 주소 입력 (회사 IP)
```

**데이터 필터 생성:**
```
관리자 → 데이터 설정 → 데이터 필터
→ "내부 트래픽 필터 만들기"
→ "제외"로 설정
→ 활성화
```

### 8.2 추천 제외 목록

**스팸 트래픽 차단:**
```
관리자 → 데이터 스트림 → 웹 스트림
→ "더보기 태그 설정"
→ "원하지 않는 추천 제외" → 켜기
```

### 8.3 데이터 보존 설정

**사용자 수준 데이터:**
```
관리자 → 데이터 설정 → 데이터 보존
→ 이벤트 데이터: 14개월 (최대)
→ 사용자 수준 데이터: 14개월
→ 재설정 ON (신규 활동 시 보존 기간 연장)
```

## 10. FAQ 및 문제 해결

**Q: 데이터가 실시간으로 안 보여요**
- 추적 코드 설치 확인 (Tag Assistant)
- 실시간 보고서에서 최소 30초 대기
- 브라우저 캐시 삭제 후 테스트

**Q: 전환 이벤트가 안 잡혀요**
- 이벤트가 정상 발생하는지 디버그 뷰 확인
- 이벤트명 철자 확인
- 전환으로 표시 설정 확인

**Q: Google Ads와 연동이 안 돼요**
- 관리자 권한 확인 (양쪽 모두)
- 자동 태깅 활성화 확인
- 연결 후 24-48시간 대기

**Q: 과거 데이터를 볼 수 있나요?**
- GA4는 설치 후 데이터만 수집
- Universal Analytics 데이터는 별도 유지
- 빅쿼리로 장기 데이터 백업 가능
