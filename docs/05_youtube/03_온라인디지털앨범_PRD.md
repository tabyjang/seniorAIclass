## 1. 개요

### 1.1 프로젝트명
**온라인 디지털 앨범** (Online Digital Album)

### 1.2 한 줄 설명
> "사진을 간편하게 올리고, 예쁘게 모아보는 나만의 디지털 앨범"

### 1.3 만드는 이유 (Why?)

**문제점:**
- 스마트폰 사진이 너무 많아서 정리가 안 됨
- 사진첩이 복잡하고 어려움
- 가족/친구와 사진 공유가 불편함
- 사진 정리하려면 컴퓨터가 필요함
- 인화해서 앨범 만드는 것이 번거로움

**해결책:**
- 웹에서 바로 사진 업로드
- 앨범별로 깔끔하게 정리
- 큰 글씨와 직관적 UI
- 링크 하나로 쉽게 공유
- 슬라이드쇼로 예쁘게 보기

**UI 참고 (ddalkak.com):**
- 메인 페이지: 상단 네비게이션 + 중앙 콘텐츠 + 그리드 갤러리
- 갤러리: 이미지 그리드 + 클릭 시 큰 이미지 보기
- 깔끔한 여백과 직관적 레이아웃

## 2. 타겟 사용자

### 2.1 주요 타겟
| 항목 | 내용 |
|------|------|
| **연령** | 50세 ~ 75세 |
| **특징** | 스마트폰/컴퓨터 기본 사용 가능 |
| **니즈** | 사진 정리, 가족 공유, 추억 보관 |
| **환경** | PC, 태블릿, 스마트폰 |

### 2.2 사용자 페르소나

**👴 김철수 (65세)**
- 손주 사진을 많이 찍음
- 딸이 해외에 살아서 사진 공유하고 싶음
- 카카오톡으로 사진 복사-붙여넣기가 어려움
- 큰 화면으로 사진 보고 싶음

**👵 박영희 (58세)**
- 여행 사진을 많이 찍음
- 동호회 회원들과 사진 공유
- 사진첩이 복잡해서 원하는 사진 찾기 어려움
- 슬라이드쇼로 보여주고 싶음

## 3. 핵심 기능 (Features)

### 3.1 필수 기능 (Must Have) ⭐

| # | 기능 | 설명 |
|---|------|------|
| 1 | **사진 업로드** | 드래그앤드롭 또는 파일 선택으로 업로드 |

| 5 | **사진 상세 보기** | 클릭 시 큰 이미지 + 정보 |
| 6 | **슬라이드쇼** | 자동 재생으로 사진 보기 |
| 7 | **공유 기능** | 링크 생성으로 공유 |
| 8 | **삭제 기능** | 사진/앨범 삭제 |

| # | 기능 | 설명 | 우선순위 |
|---|------|------|---------|

## 5. 사용자 흐름 (User Flow)

### 5.1 사진 업로드부터 공유까지

```
1. 메인 페이지 진입
   ↓
2. [+ 새 앨범] 클릭
   ↓
3. 앨범 정보 입력 (이름, 설명, 날짜)
   ↓
4. 사진 업로드 (드래그 또는 선택)
   ↓
5. 업로드 완료 → 앨범으로 이동
   ↓
6. 사진 확인 (그리드 형태)
   ↓
7. 클릭해서 상세 보기
   ↓
8. [공유] 버튼 클릭
   ↓
9. 링크 복사 또는 전송
```

### 5.2 슬라이드쇼 보기

```
1. 앨범 진입
   ↓
2. [▶️ 슬라이드쇼 보기] 클릭
   ↓
3. 전체화면으로 슬라이드쇼 시작
   ↓
4. 3초마다 다음 사진 자동 전환
   ↓
5. 클릭하면 일시정지/재생
   ↓
6. ESC 또는 닫기 버튼으로 종료
```

**1. Masonry 레이아웃:**

- 사진 비율 유지 (object-fit: cover)
- 최소 너비 250px, 최대 300px

```css
  justify-content: center;

```
