# 수강결과 페이지 (results.html) 기획서

> **작성일:** 2026-02-04
> **목적:** 수강생들이 만든 프로덕트 결과물을 보여주는 갤러리 페이지

---

## 1. 페이지 목적

### 핵심 가치
- **신뢰 구축**: "이 수업을 들으면 이런 결과물을 만들 수 있다"
- **동기 부여**: 다른 수강생들의 성과를 보고 "나도 할 수 있다" 느끼기
- **증명**: 40개 프로덕트가 실제로 완성된다는 증거

### 타겟 사용자
1. **예비 수강생**: 수업 결과물 미리보기 → 등록 유도
2. **현재 수강생**: 다른 사람들 결과물 보며 자극받기
3. **수료생**: 자신의 포트폴리오 공유

## 3. 기능 명세

### 3.1 필터링
| 기능 | 설명 |
|------|------|
| 레벨 필터 | Lv.1~4 탭으로 필터링 |
| 프로덕트 필터 | 40개 프로덕트 중 선택 |
| 정렬 | 최신순, 인기순, 레벨순 |

### 3.2 데이터 구조
```javascript
{
  id: 1,
  productId: 1,                    // 프로덕트 번호 (1~40)
  productName: "나의 첫 Gmail",
  level: 1,
  studentName: "김OO",             // 익명화된 이름
  studentAge: 62,
  completedDate: "2026-01-15",
  thumbnails: ["url1", "url2"],
  review: "처음 만든 이메일...",
  criteria: ["@gmail.com 생성", "프로필 설정", "첫 이메일 발송"],
  likes: 45
}
```

### 3.3 인터랙션
1. **카드 hover**: 살짝 확대 + 그림자
2. **카드 클릭**: 상세 모달 열기
3. **필터 변경**: 애니메이션과 함께 그리드 재배치
4. **무한 스크롤 or 더보기**: 추가 로딩

## 5. 기술 구현

### 5.1 선택지
| 옵션 | 장점 | 단점 |
|------|------|------|
| **A. 독립 HTML** | 빠른 구현, 간단 | React 컴포넌트 재사용 불가 |
| **B. React 컴포넌트** | 기존 시스템 활용 | 빌드 필요 |

**선택: A. 독립 HTML**
- 이유: 빠른 배포, 정적 데이터로 충분

### 5.2 데이터 소스
- **초기**: 하드코딩 (샘플 데이터 10~20개)
- **향후**: JSON 파일 또는 CMS 연동

### 5.3 반응형 그리드
| 화면 | 컬럼 수 |
|------|---------|
| 모바일 (< 640px) | 1열 |
| 태블릿 (640px~) | 2열 |
| 데스크톱 (1024px~) | 3열 |
| 와이드 (1280px~) | 4열 |
