# 프로덕트 문서화 가이드

## 🚀 시작 가이드 (디폴트 템플릿)

```markdown
# {제품명} - 초보자용 시작 가이드

**작성일:** YYYY-MM-DD
**난이도:** 완전 초보자용  
**목표:** 처음부터 끝까지 따라하면 {제품명}이 동작하게 만들기

---

## 📋 이 문서 사용법

이 가이드는 프로덕트 개발의 첫걸음을 떼는 초보자를 위한 로드맵입니다.  
각 단계별로 무엇을 만들고, 어떻게 검증하는지 명확히 제시합니다.

**진행 순서:**
1. PRD 읽기 - 뭘 만들지 이해하기
2. 기술문서 확인 - 어떻게 만들지 확인
3. 로드맵 따라 개발 - 단계별로 진행
4. 체크리스트로 검증 - 완성도 체크

**완성 기준:**
- [ ] 모든 체크리스트 항목 완료
- [ ] 실제 동작하는 제품
- [ ] 사용자가 직접 테스트 가능
```

---

## 문서 작성 순서

```
1️⃣ PRD → 2️⃣ 기술문서 → 3️⃣ 로드맵 → 4️⃣ 체크리스트
```

**초보자용 팁:** 문서 4개를 한꺼번에 작성하지 마세요. 순서대로 하나씩 작성하며 진행하세요.

| 순서 | 문서 | 핵심 질문 | 작성 시점 |
|------|------|----------|----------|
| 1️⃣ | PRD | "무엇을 만들까?" | 기획 단계 |
| 2️⃣ | TECH | "어떻게 만들까?" | 설계 단계 |
| 3️⃣ | ROADMAP | "언제까지 만들까?" | 계획 단계 |
| 4️⃣ | CHECKLIST | "뭘 했고 뭐가 남았나?" | 개발 단계 |

## 1️⃣ PRD (Product Requirements Document)

### 🎯 초보자용 시작 가이드

**이 단계에서 할 일:**
- 제품이 무엇인지 한 문장으로 정의하기
- 누구를 위해 만드는지 명확히 하기
- 핵심 기능 3가지만 먼저 정하기

**완료 체크:**
- [ ] 제품명이 정해졌다
- [ ] 한 줄 소개를 작성했다  
- [ ] 타겟 사용자를 구체적으로 적었다
- [ ] 핵심 기능 3가지를 나열했다

**초보자 팁:**  
처음부터 완벽할 필요 없습니다. 대략적인 아이디어만 적어도 충분해요. 개발하면서 자연스럽게 구체화됩니다.

### 목적
제품이 **무엇**인지, **왜** 만드는지, **누구**를 위한 것인지 정의

### 필수 섹션
```markdown
# {제품명} PRD

## 1. 제품 개요
- 제품명
- 한 줄 소개
- 타겟 사용자
- 핵심 문제 & 솔루션

## 2. 기능 요구사항
- 핵심 기능 (MVP)
- 추가 기능 (v2)

## 3. UI/UX 설계
- 화면 구성
- 색상/폰트

## 4. 시스템 요구사항
- OS, 메모리 등

## 5. 성공 지표
- KPI, 목표
```

## 2️⃣ TECH (기술 설계서)

### 🎯 초보자용 시작 가이드

**이 단계에서 할 일:**
- 어떤 기술을 사용할지 정하기 (복잡하게 생각하지 마세요)
- 파일 구조 간단히 그려보기
- 핵심 기능 구현 방법 찾기 (구글 검색 OK)

**완료 체크:**
- [ ] 사용할 언어/프레임워크를 정했다
- [ ] 전체 파일 구조를 그려봤다
- [ ] 핵심 기능을 어떻게 만들지 구체적으로 적었다
- [ ] 필요한 라이브러리를 나열했다

**초보자 팁:**  
코드 예시는 검색해서 찾아도 됩니다. "파이썬 QR 코드 생성" 이렇게 검색하면 됩니다. 완벽하게 이해하지 못해도 일단 따라해보세요.

### 목적
제품을 **어떻게** 만들지 기술적으로 설계

### 필수 섹션
```markdown
# {제품명} 기술 설계서

## 1. 아키텍처 개요
- 시스템 구조도
- 모듈 관계

## 2. 기술 스택
- 언어, 프레임워크, 라이브러리

## 3. 핵심 기술
- 주요 알고리즘/API 설명
- 코드 예시

## 4. 모듈/클래스 설계
- 주요 클래스 구조
- 메서드 설명

## 5. 데이터 구조
- 설정 파일 형식
- 저장소 구조

## 6. 에러 처리
- 예외 상황 대응

## 7. 테스트 계획
- 테스트 항목
```

## 3️⃣ ROADMAP (로드맵)

### 🎯 초보자용 시작 가이드

**이 단계에서 할 일:**
- 최소한의 기능(MVP)을 먼저 정하기 (완벽하지 않아도 됨)
- 하루에 얼마나 할 수 있는지 현실적으로 계획하기
- 우선순위 높은 것부터 순서대로 나열하기

**완료 체크:**
- [ ] MVP(최소 기능)를 3가지 이내로 정했다
- [ ] 각 단계별로 몇 시간/며칠이 필요한지 예상했다
- [ ] 순서대로 진행할 작업 목록을 만들었다
- [ ] 첫 번째 작업부터 시작할 수 있다

**초보자 팁:**  
하루에 1-2시간만 투자해도 충분합니다. 완벽한 계획보다 "일단 시작"이 중요해요. 로드맵은 개발하면서 수정해도 됩니다.

### 목적
**언제까지** 무엇을 완성할지 일정 계획

### 필수 섹션
```markdown
# {제품명} 로드맵

## Phase 1: MVP
- 목표
- 기능 목록
- 예상 기간

## Phase 2: 개선
- 목표
- 기능 목록
- 예상 기간

## Phase 3: 배포
- 목표
- 작업 목록

## 마일스톤 요약
| 버전 | 목표 | 예상 완료 |
|------|------|----------|
| v0.1 | MVP | Day 1 |
| v1.0 | 정식 | Day 5 |
```

## 4️⃣ CHECKLIST (체크리스트)

### 🎯 초보자용 시작 가이드

**이 단계에서 할 일:**
- 로드맵에서 정한 작업을 하나씩 실행하기
- 완료된 항목은 즉시 체크하기 (성취감 중요!)
- 막히는 부분은 메모에 적고 다음으로 넘어가기
- 매일 1시간이라도 진행 상황 업데이트하기

**완료 체크:**
- [ ] 오늘 할 작업을 1-3개로 줄였다
- [ ] 완료된 작업에 체크를 했다
- [ ] 문제가 생긴 부분을 메모에 적었다
- [ ] 내일 할 작업을 미리 정했다

**초보자 팁:**  
체크리스트는 매일 업데이트하세요. 한 칸이라도 채우면 성취감이 생깁니다. 완벽하지 않아도 "됐다!" 하고 체크하세요. 70% 완성해도 훌륭한 결과물입니다.

### 목적
실제 개발 진행 상황 추적

### 필수 섹션
```markdown
# {제품명} 개발 체크리스트

## 현재 상태
- 🟢 완료: X개
- 🟡 진행중: X개
- ⚪ 대기: X개

## Phase 1: MVP
- [x] 완료된 태스크
- [ ] 미완료 태스크

## Phase 2: 개선
- [ ] 태스크 1
- [ ] 태스크 2

## 이슈/메모
- 발생한 문제와 해결책
```

## ✅ 완성 기준 (초보자용)

### 프로덕트 완성의 정의

**"완성"은 100% 완벽할 필요가 없습니다.**

**최소 완성 기준:**
- [ ] PRD에 적힌 핵심 기능 3가지가 동작한다
- [ ] 사용자가 직접 사용해볼 수 있다
- [ ] 치명적인 오류가 없다 (작은 버그는 OK)
- [ ] 체크리스트 70% 이상 완료

**더 나아가기 (선택):**
- [ ] 디자인을 다듬는다
- [ ] 추가 기능을 넣는다
- [ ] 문서를 보강한다

### 초보자의 마인드셋

```
✓ "일단 돌아가게 만들자" (MVP 우선)
✓ "완벽하지 않아도 된다" (70% = 완성)
✓ "매일 조금씩 진행하자" (1시간/일 OK)
✓ "막히면 검색하자" (구글/ChatGPT 활용)
✓ "체크하면 성취감을 느끼자" (동기부여 중요!)
```

### 문제 해결 팁

**막혔을 때:**
1. 30분만 더 시도해보기
2. 에러 메시지를 구글에 검색하기
3. ChatGPT에게 "~가 안 돼요, 왜 그럴까요?" 물어보기
4. 로드맵에서 다음 작업으로 잠시 넘어가기
5. 쉬고 내일 다시 도전하기 (피로하면 해결 안 됨)

**도움 요청하기:**
- 동료나 커뮤니티에 질문하기
- "시니어AI클래스" 디스코드/카톡방 이용하기
- 강사에게 멘토링 요청하기
