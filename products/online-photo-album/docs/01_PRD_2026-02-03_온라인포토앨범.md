## 1. 제품 개요

### 1.1 제품명
**온라인 포토 앨범** (Online Photo Album)

### 1.2 한 줄 소개
> "사진을 올리고, 예쁘게 모아보고, 쉽게 공유하는 나만의 디지털 앨범"

### 1.3 타겟 사용자
- **연령:** 50-75세 시니어
- **특징:** 스마트폰/컴퓨터 기본 사용 가능
- **니즈:** 사진 정리, 가족 공유, 추억 보관
- **환경:** PC, 태블릿, 스마트폰

### 1.4 핵심 문제
| 문제 | 상세 |
|------|------|
| 사진 정리 어려움 | 스마트폰 사진이 너무 많아서 찾기 힘듦 |
| 공유 불편 | 카톡으로 사진 여러 장 복사-붙여넣기 복잡 |
| 앨범 앱 어려움 | 기존 앱은 기능이 많아서 복잡함 |
| 인화 번거로움 | 사진관 가서 인화하는 것이 귀찮음 |

### 1.5 솔루션
| 해결책 | 방법 |
|---------|------|
| 간단한 업로드 | 드래그앤드롭 또는 파일 선택 |
| Pinterest 스타일 갤러리 | Masonry 레이아웃으로 예쁘게 배열 |
| 앨범별 정리 | 주제별로 폼더처럼 관리 |
| 링크 공유 | 한 클릭으로 공유 링크 생성 |
| 슬라이드쇼 | 전체화면으로 예쁘게 보기 |

## 2. 핵심 기능

### 2.1 필수 기능 (MVP)

| # | 기능 | 설명 |
|---|------|------|
| 1 | **사진 업로드** | 드래그앤드롭 또는 클릭으로 파일 선택 |
| 2 | **앨범 생성** | 이름, 설명, 날짜 설정 |
| 3 | **앨범 목록** | 카드 형태로 앨범 미리보기 |
| 4 | **Masonry 갤러리** | Pinterest 스타일 그리드 (높이 다른 사진 자연스럽게 배열) |
| 5 | **사진 상세 보기** | 클릭 시 라이트박스로 큰 이미지 |
| 6 | **슬라이드쇼** | 자동 재생으로 사진 보기 |
| 7 | **공유 링크** | 링크 복사로 쉽게 공유 |
| 8 | **사진 삭제** | 개별/선택 삭제 |

### 2.2 추가 기능 (v2)

| # | 기능 | 설명 | 우선순위 |
|---|------|------|---------|
| 1 | **사진 설명** | 사진별 메모/날짜/장소 추가 | 중 |
| 2 | **즐겨찾기** | 중요한 사진 하트 표시 | 중 |
| 3 | **다운로드** | 사진 개별/전체 다운로드 | 중 |
| 4 | **정렬** | 날짜/이름 순 정렬 | 하 |
| 5 | **검색** | 앨범/사진 검색 | 하 |

### 2.3 제외 기능
| 기능 | 제외 이유 |
|------|----------|
| 회원가입/로그인 | 익명으로 바로 사용 |
| 사진 편집 | 복잡함, 외부 앱 사용 유도 |
| 댓글/좋아요 | 커뮤니티 기능 불필요 |
| 인화 주문 | 별도 서비스 연동 필요 |
| 클라우드 저장 | 로컬 브라우저 저장으로 시작 |

## 3. UI/UX 설계

### 3.1 디자인 컨셉: Pinterest 스타일

**핵심 요소:**
1. **Masonry 레이아웃**: 세로/가로 사진이 자연스럽게 어우러짐
2. **카드 UI**: 이미지 + 메타정보(날짜/장소/설명)
3. **클린 디자인**: 여백 충분, 직관적 아이콘
4. **호버 효과**: 마우스 올리면 액션 버튼 표시

**Masonry 특징:**
- 사진 원본 비율 유지 (4:3, 3:4, 1:1 등)
- 2~4열 반응형 (화면 크기에 따라 자동 조정)
- 사진 사이 16px 여백
- 둥근 모서리 (border-radius: 16px)

### 3.4 사진 카드 상세

```
┌─────────────────────┐
│                     │
│    사진 (원본 비율)  │
│                     │
│  ┌───────────────┐  │
│  │   [🔍 보기]   │  │ ← 호버 시 표시
│  └───────────────┘  │
├─────────────────────┤

│ "사진 설명 메모"    │
└─────────────────────┘
```

**호버 효과:**
- 어두운 오버레이 (30% 검정)
- 중앙에 [🔍 크게 보기] 버튼
- 우측 상단 [❤️] 즐겨찾기 버튼
- 부드러운 전환 (0.3s ease)

## 5. 기술 요구사항

### 5.1 기술 스택

| 구분 | 기술 | 이유 |
|------|------|------|
| 프론트엔드 | HTML/CSS/JavaScript | 단순, 빠름, 호환성 |
| 레이아웃 | CSS Columns | Pinterest 스타일 Masonry |
| 데이터 저장 | localStorage / IndexedDB | 서버 없이 브라우저 저장 |
| 이미지 처리 | Canvas API | 리사이즈, 썸네일 생성 |
| 배포 | GitHub Pages | 무료, 쉬움 |

### 5.2 Masonry 레이아웃 구현

**CSS Columns 방식 (권장):**
```css
.gallery {
  column-count: 4;
  column-gap: 16px;
}

.photo-card {
  break-inside: avoid;
  margin-bottom: 16px;
  border-radius: 16px;
  overflow: hidden;
}

/* 반응형 */
@media (max-width: 1200px) { .gallery { column-count: 4; } }
@media (max-width: 900px) { .gallery { column-count: 3; } }
@media (max-width: 600px) { .gallery { column-count: 2; } }
```
