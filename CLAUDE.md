# 프로젝트 작업 지침

## 필수 규칙

### 1. 문서화 우선
- **분석, 계획, 체크리스트 등은 반드시 먼저 .md 문서로 저장한 후 설명할 것**
- 대화가 길어지면 컨텍스트 압축으로 상세 내용이 사라짐
- 중요한 내용은 항상 파일로 남겨야 함

### 2. 문서 작성 규칙
- 파일명 형식: `YYYY-MM-DD_제목.md`
- 저장 위치: `docs/` 폴더
- 문서 상단에 날짜, 제목, 목적 명시

### 3. 프로덕트 문서화 규칙 (필수!)
프로덕트 개발 시 반드시 아래 순서와 형식으로 문서 작성:

```
1️⃣ PRD → 2️⃣ TECH → 3️⃣ ROADMAP → 4️⃣ CHECKLIST
```

**파일명 형식:**
```
{순번}_{문서유형}_{날짜}_{제품명}.md
```

**예시:**
```
products/{제품명}/docs/
├── 01_PRD_2026-02-02_고스트프롬프트.md      # 무엇을 만들까?
├── 02_TECH_2026-02-02_고스트프롬프트.md     # 어떻게 만들까?
├── 03_ROADMAP_2026-02-02_고스트프롬프트.md  # 언제까지 만들까?
└── 04_CHECKLIST_2026-02-02_고스트프롬프트.md # 진행 상황 추적
```

**상세 가이드:** `senioraiclass/docs/PRODUCT_DOC_GUIDE.md` 참고

### 4. 언어
- 한국어로 응답
- 코드 주석도 한국어
